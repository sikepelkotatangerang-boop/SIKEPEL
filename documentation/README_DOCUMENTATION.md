## üìù Documentation Standards

### File Naming
- **UPPERCASE**: Major documentation (e.g., `SETUP.md`, `PROJECT_RULES.md`)
- **kebab-case**: Feature-specific docs (e.g., `user-management.md`)

### Structure
All documentation should include:
1. Title
2. Overview/Purpose
3. Main content with clear sections
4. Examples (if applicable)
5. Related documentation links
6. Last updated date

### Location
**ALL** `.md` files MUST be in `documentation/` folder (except root `README.md`).

See [DOCUMENTATION_ORGANIZATION.md](./DOCUMENTATION_ORGANIZATION.md) for details.
